    2. Add a stub `InviteHandler` component near the top of the file (before `WorkspaceSwitcher`):

       This stub will be replaced entirely in Plan 09-03. It just ensures the route renders without crashing.

    3. Replace the `export default function App()` body so it wraps everything in BrowserRouter with two routes:
       ```jsx
       export default function App() {
         return (
           <BrowserRouter>
             <Routes>
               <Route
                 path="/invite"
                 element={
                   <AuthProvider>
                     <InviteHandler />
                   </AuthProvider>
                 }
               />
               <Route
                 path="/*"
                 element={
                   <AuthProvider>
                     <AuthGate>
                       <WorkspaceProvider>
                         <WorkspaceGate />
                       </WorkspaceProvider>
                     </AuthGate>
                   </AuthProvider>
                 }
               />
             </Routes>
           </BrowserRouter>
         );
       }
       ```

    IMPORTANT notes:
    - The `/invite` route gets its own `AuthProvider` so it has access to `useAuth()` for the flows in Plan 09-03, but does NOT go through `AuthGate` — unauthenticated users must reach it.
    - `path="/*"` (wildcard) catches root and any other paths, routing them to the existing app stack. This is intentional — preserves all existing behaviour.
    - Do NOT move `WorkspaceSwitcher`, `AppContent`, `WorkspaceGate` out of App.jsx — they stay exactly where they are.
    - Keep all existing imports intact (AuthProvider, AuthGate, WorkspaceProvider, etc.).
  </action>
