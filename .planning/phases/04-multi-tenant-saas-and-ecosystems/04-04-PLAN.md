<objective>
Split the recipe pipeline into two steps: AI extraction (preview) and user-confirmed save. The client shows an editable form between the two steps.

Purpose: Allows users to fix AI errors (wrong title, wrong category, missing ingredient) before the recipe is persisted.
Output: Two new server endpoints (/api/extract-recipe, /api/confirm-recipe) and a RecipeEditForm component that appears between URL submission and gallery save.
</objective>

    Add two new route handlers to `server/src/routes/recipe.js` (keep existing /api/process-recipe unchanged):

    **POST /api/extract-recipe:**
    - Accepts `{ instagram_url }` from request body
    - Runs scrapeInstagramCaption + extractRecipeFromCaption (same as process-recipe steps 1+2)
    - Returns the extracted recipe WITHOUT saving: `{ success: true, instagram_url, title, main_category, difficulty, ingredients }` on 200
    - Error handling identical to process-recipe (abort detection, 422 vs 500, error message)
    - Does NOT call saveRecipe

    **POST /api/confirm-recipe:**
    - Accepts `{ instagram_url, title, main_category, difficulty, ingredients, workspace_id }` from request body (user-edited values)
    - Validates: instagram_url required, title required, ingredients must be an array
    - Calls `saveRecipe({ instagram_url, title, main_category, difficulty, ingredients, workspace_id })`
    - Returns `{ success: true, recipe_id, title, ingredients_count, workspace_id }` on 200
    - Returns 400 with `{ success: false, error }` on validation failure
    - Returns 500 with `{ success: false, error }` on save failure
  </action>
