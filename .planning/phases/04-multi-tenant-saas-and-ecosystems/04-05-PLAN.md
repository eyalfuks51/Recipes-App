The `workspace_ingredient_checks` table was created in plan 04-02 with schema:
  PRIMARY KEY (workspace_id, recipe_id, ingredient_id)
  checked BOOLEAN NOT NULL DEFAULT false

For v1 of workspaces: workspace_id is optional (recipes may have NULL workspace_id). When workspace_id is null, checkbox state is still stored using a "personal" workspace keyed to the user's auth.uid. Approach: if recipe has no workspace_id, use `user_id` in place of workspace_id for the checks table lookup (treat personal use as a single-user workspace).

RecipeGallery currently fetches `id, title, main_category, difficulty, instagram_url` from recipes. Need to also fetch `instructions` and `workspace_id`.

RecipeModal currently shows ingredient list as static bullet items. Each `<li>` needs to become a `<label>` with `<input type="checkbox">`.

The ingredient list in modal is fetched as `{ ingredients(name) }` via recipe_ingredients join. For checkbox state we also need ingredient `id` — update the join query.

    ```js
    const { user } = useAuth();
    const [checkedIds, setCheckedIds] = useState(new Set());
    const [checksLoading, setChecksLoading] = useState(false);

    // Effective workspace key: use recipe.workspace_id if set, else user.id (personal mode)
    const workspaceKey = recipe.workspace_id ?? user?.id ?? null;
    ```

    4. Load existing checkbox state when ingredients are ready (useEffect on `[ingredients, workspaceKey]`):
    ```js
    useEffect(() => {
      if (!workspaceKey || ingredients.length === 0 || ingredientsLoading) return;
      setChecksLoading(true);
      supabase
        .from('workspace_ingredient_checks')
        .select('ingredient_id')
        .eq('workspace_id', workspaceKey)
        .eq('recipe_id', recipe.id)
        .eq('checked', true)
        .then(({ data }) => {
          if (data) setCheckedIds(new Set(data.map((r) => r.ingredient_id)));
          setChecksLoading(false);
        });
    }, [ingredients, workspaceKey, ingredientsLoading]);
    ```

    5. Add toggle handler:
    ```js
    async function handleToggle(ingredientId) {
      if (!workspaceKey) return;
      const nowChecked = !checkedIds.has(ingredientId);
      // Optimistic update
      setCheckedIds((prev) => {
        const next = new Set(prev);
        nowChecked ? next.add(ingredientId) : next.delete(ingredientId);
        return next;
      });
      // Persist to Supabase (upsert)
      await supabase.from('workspace_ingredient_checks').upsert({
        workspace_id: workspaceKey,
        recipe_id: recipe.id,
        ingredient_id: ingredientId,
        checked: nowChecked,
        updated_at: new Date().toISOString(),
      }, { onConflict: 'workspace_id,recipe_id,ingredient_id' });
    }
    ```

    6. Update ingredient list rendering (replace static `<li>` with checkbox `<label>`):
    ```jsx
    {!ingredientsLoading && ingredients.length > 0 && (
      <ul className="ingredient-list">
        {ingredients.map((ingredient) => (
          <li key={ingredient.id} className={`ingredient-item ${checkedIds.has(ingredient.id) ? 'ingredient-item--checked' : ''}`}>
            <label className="ingredient-label">
              <input
                type="checkbox"
                checked={checkedIds.has(ingredient.id)}
                onChange={() => handleToggle(ingredient.id)}
                className="ingredient-checkbox"
              />
              <span>{capitalize(ingredient.name)}</span>
            </label>
          </li>
        ))}
      </ul>
    )}
    ```
