<objective>
Build the RecipeReviewScreen — a premium split-screen component that shows the original Instagram post alongside an editable form with the full multi-dimensional AI output — and wire it into SubmitForm.

Purpose: Users need to review and verify AI-extracted recipe data before it hits the database. The existing RecipeEditForm is too minimal for this workflow — it lacks the Instagram post preview and all new fields. This new dedicated screen delivers the "inbox" experience where users see the source post while correcting each field.

Output: RecipeReviewScreen.jsx + RecipeReviewScreen.scss (new files); SubmitForm.jsx updated to render the new screen instead of RecipeEditForm on preview state.
</objective>

Create `client/src/components/RecipeReviewScreen.jsx` — a new component replacing RecipeEditForm in the preview step.

**State to manage:**
- `title` (string)
- `category` (string) — from extractedRecipe.main_category
- `difficulty` (string)
- `ingredientsText` (string) — newline-joined ingredient array (same as current form)
- `steps` (string[]) — extracted from extractedRecipe.instructions array; each element is one step string
- `mealType` (string)
- `cuisine` (string)
- `mainIngredient` (string)
- `equipment` (string[]) — array of equipment strings
- `prepTime` (string) — controlled number input stored as string, parsed to int on save
- `cookTime` (string)
- `dietaryTags` (string[]) — array of selected tag strings
- `saving` (boolean)
- `saveError` (string | null)
- `activeTab` (string) — `'post'` or `'edit'`, only relevant on mobile

**Instagram embed (reuse RecipeModal.jsx pattern):**
```javascript
function extractShortcode(url) {
  const match = url?.match(/instagram\.com\/(?:p|reel|tv)\/([A-Za-z0-9_-]+)/);
  return match?.[1] ?? null;
}
```
Use this to render the iframe embed in the left panel.

**Instructions section UI:**
- Render each step as a numbered row: `<span className="step-number">{index + 1}</span>` + `<input type="text" value={step} />`
- "מחק" (delete) button per step — removes from steps array
- "+ הוסף שלב" button at the bottom — appends empty string to steps array
- Step numbers are display-only; not stored in the text value

**Dietary tags UI:**
- Render ALLOWED_DIETARY_TAGS as checkboxes
- `checked={dietaryTags.includes(tag)}`
- On change: toggle tag in/out of dietaryTags array

**Equipment needed UI:**
- Render each equipment string as a tag with an "x" remove button
- Text input + "הוסף" button to add new items to the equipment array

**handleSave:**
- Parse ingredientsText to string[] (split by newline, trim, filter empty)
- Parse prepTime/cookTime to integer or null (use `parseInt(val) || null`)
- POST to `/api/confirm-recipe` with all fields
- On success: call `onSaved(data)`
- On error: set saveError

**Layout (JSX structure):**
