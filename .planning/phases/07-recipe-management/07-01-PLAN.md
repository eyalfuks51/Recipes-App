  <name>Task 1: Add deleteRecipe and updateRecipe to supabase.js</name>
  <files>server/src/services/supabase.js, server/src/__tests__/supabase.test.js</files>
  <behavior>
    - deleteRecipe('existing-id'): deletes row from `recipes` table by id, returns { deleted: true }
    - deleteRecipe('nonexistent-id'): Supabase returns count 0 → throw Error('Recipe not found')
    - deleteRecipe propagates Supabase errors as thrown Errors
    - updateRecipe('existing-id', fields): upserts all provided fields using .update().eq('id', id).select(), returns { recipe_id, title }
    - updateRecipe('nonexistent-id', fields): Supabase returns empty data array → throw Error('Recipe not found')
    - updateRecipe propagates Supabase errors as thrown Errors
    - Ingredients: updateRecipe also receives `ingredients` string array; re-upserts into `ingredients` table and replaces `recipe_ingredients` junction rows (delete old junctions, insert new ones)
  </behavior>
  <action>
    Add two exported async functions to server/src/services/supabase.js after saveRecipe:

    `deleteRecipe(id)`:
    - Call `client.from('recipes').delete().eq('id', id).select('id')`
    - If error → throw new Error(`Recipe delete failed: ${error.message}`)
    - If data is empty array → throw new Error('Recipe not found')
    - Return `{ deleted: true }`

    `updateRecipe(id, { title, main_category, difficulty, ingredients, workspace_id, instructions, meal_type, cuisine, main_ingredient, prep_time, dietary_tags, thumbnail_url })`:
    - Build update object with only provided fields (same spread pattern as saveRecipe's recipeData)
    - Call `client.from('recipes').update(recipeData).eq('id', id).select('id, title, workspace_id')`
    - If error → throw new Error(`Recipe update failed: ${error.message}`)
    - If data is empty array → throw new Error('Recipe not found')
    - If `ingredients` array provided: upsert ingredients table (same pattern as saveRecipe step 2), then delete existing recipe_ingredients rows for this recipe_id with `.delete().eq('recipe_id', id)`, then insert fresh junction rows
    - Return `{ recipe_id: recipe.id, title: recipe.title, workspace_id: recipe.workspace_id ?? null }`

    Add tests for both functions in server/src/__tests__/supabase.test.js following the existing mock pattern (global.__mockSupabaseClient). The mock client needs to handle: `.from('recipes').delete().eq().select()` and `.from('recipes').update().eq().select()` chain shapes. Test success path and not-found path for each.
  </action>
  <verify>
    <automated>cd C:/dev/github/personal/recepies_api/server && node --test src/__tests__/supabase.test.js</automated>
  </verify>
  <done>
