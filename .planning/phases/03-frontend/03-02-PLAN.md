# Plan 03-02: URL Submission Form

## Objective

Build the `SubmitForm` component that accepts an Instagram URL, POSTs it to the backend, and shows loading / success / error feedback inline.

## Context

Depends on 03-01 (Vite scaffold exists, env vars documented). The backend endpoint is `POST /api/process-recipe` accepting `{ instagram_url }` and returning `{ success, recipe_id, title, ingredients_count }` on success or `{ success: false, error: string }` on failure. The backend URL comes from `import.meta.env.VITE_API_URL`.

The `App` component renders `SubmitForm` and passes an `onSuccess` callback so the gallery (plan 03-03) can refresh when a new recipe is saved. Wiring in `App.jsx` is minimal here â€” plan 03-03 completes the connection.

<interfaces>
Backend contract (from server/src/routes/recipe.js):

POST {VITE_API_URL}/api/process-recipe
Request body: { instagram_url: string }

Success response (200): { success: true, recipe_id: string, title: string, ingredients_count: number }
Error response (400/422/500): { success: false, error: string }
</interfaces>

## Tasks

<task id="1">
### Task 1: Build SubmitForm component with full state machine

**Action:** Create `client/src/components/SubmitForm.jsx` implementing the URL input, submit handler, and three feedback states: loading, success, error.

**Files:**
- `client/src/components/SubmitForm.jsx`

**Details:**

Component interface:
```jsx
// Props:
// - onSuccess: (recipe: { recipe_id, title, ingredients_count }) => void
//   Called after a successful submit so the gallery can refresh.
export function SubmitForm({ onSuccess }) { ... }
```

State:
- `url` (string) â€” controlled input value
- `status` â€” one of `'idle' | 'loading' | 'success' | 'error'`
- `result` â€” on success: `{ title, ingredients_count }`, on error: `{ error: string }`

Submit handler logic:
1. `preventDefault` on the form submit event
2. Set `status = 'loading'`
3. `fetch(`${import.meta.env.VITE_API_URL}/api/process-recipe`, { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ instagram_url: url }) })`
4. Parse JSON response
5. If `response.ok` and `data.success`: set `status = 'success'`, `result = { title: data.title, ingredients_count: data.ingredients_count }`, call `onSuccess(data)`
6. If not ok or `data.success === false`: set `status = 'error'`, `result = { error: data.error || 'Unknown error' }`
7. In catch block: set `status = 'error'`, `result = { error: err.message }`

Render:
- `<form onSubmit={handleSubmit}>` wrapping the whole form
- `<input type="url" placeholder="https://www.instagram.com/p/..." value={url} onChange={...} required disabled={status === 'loading'} />`
- `<button type="submit" disabled={status === 'loading'}>` showing text `"Submit"` normally, `"Processing..."` when loading
- When `status === 'success'`: render `<p className="success-msg">Saved: {result.title} ({result.ingredients_count} ingredients)</p>`
- When `status === 'error'`: render `<p className="error-msg">{result.error}</p>`
- Wrap feedback in a `<div className="form-feedback">` below the input row

Do NOT reset the url input after success â€” user may want to inspect the URL.
</task>
