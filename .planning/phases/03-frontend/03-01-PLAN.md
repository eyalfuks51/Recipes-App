# Plan 03-01: Vite/React Project Scaffold

## Objective

Bootstrap the `/client` Vite + React project with SCSS support, environment variable wiring, and Vercel SPA deployment config — everything downstream plans build on.

## Context

The project is a monorepo: `/server` (Express, deployed to Koyeb) and `/client` (React/Vite, deployed to Vercel). This plan creates `/client` from scratch. No `/client` directory exists yet. All subsequent frontend plans depend on this scaffold.

## Tasks

<task id="1">
### Task 1: Scaffold Vite React project and install dependencies

**Action:** From the repo root (`/`), create the `/client` directory and initialise a Vite + React project manually (do NOT use `npm create vite@latest` interactively — write files directly). Install all required dependencies.

**Files:**
- `client/package.json`
- `client/index.html`
- `client/vite.config.js`
- `client/src/main.jsx`
- `client/src/App.jsx`

**Details:**

`client/package.json`:
```json
{
  "name": "recepies-client",
  "version": "1.0.0",
  "private": true,
  "type": "module",
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview"
  },
  "dependencies": {
    "@supabase/supabase-js": "^2.49.1",
    "react": "^18.3.1",
    "react-dom": "^18.3.1"
  },
  "devDependencies": {
    "@vitejs/plugin-react": "^4.3.4",
    "sass": "^1.86.0",
    "vite": "^6.2.0"
  }
}
```

`client/vite.config.js`:
```js
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';

export default defineConfig({
  plugins: [react()],
});
```

`client/index.html` — standard Vite HTML entry with `<div id="root">` and `<script type="module" src="/src/main.jsx">`.

`client/src/main.jsx` — renders `<App />` into `#root`, imports `./styles/main.scss`.

`client/src/App.jsx` — minimal shell: exports a default `App` component returning `<div className="app"><h1>Recipe Manager</h1></div>`. Will be expanded in plans 03-02 and 03-03.

After writing files, run `npm install` inside `client/`:
```bash
cd C:/dev/github/personal/recepies_api/client && npm install
```
</task>

<task id="2">
### Task 2: Add SCSS entry, env example, and Vercel SPA config

**Action:** Create the SCSS root file, document required env vars, and add the Vercel rewrite rule so Vite SPA routes work after deployment.

**Files:**
- `client/src/styles/main.scss`
- `client/.env.example`
- `client/vercel.json`

**Details:**

`client/src/styles/main.scss` — root stylesheet imported by `main.jsx`. Include a CSS reset and base typography:
```scss
*, *::before, *::after {
  box-sizing: border-box;
  margin: 0;
  padding: 0;
}

body {
  font-family: system-ui, sans-serif;
  background: #f5f5f5;
  color: #1a1a1a;
  line-height: 1.5;
}

.app {
  max-width: 900px;
  margin: 0 auto;
  padding: 2rem 1rem;
}
```

`client/.env.example` — documents all frontend env vars (these are the only ones Vite exposes via `import.meta.env`):
```
# Backend API (deployed Koyeb URL or localhost for dev)
VITE_API_URL=https://your-koyeb-app.koyeb.app

# Supabase — read-only gallery access (anon key is safe to expose)
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key-here
```

`client/vercel.json` — SPA rewrite rule so direct URL navigation works on Vercel:
```json
{
  "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }]
}
```

No `.env` file should be committed. The executor should create a real `client/.env` locally (not committed) with actual values for dev testing.
</task>
