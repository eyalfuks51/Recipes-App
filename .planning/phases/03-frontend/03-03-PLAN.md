# Plan 03-03: Recipe Gallery with Supabase Reads

## Objective

Build the `RecipeGallery` component that reads recipes directly from Supabase on page load, renders each as a card (title + category + difficulty), and refreshes automatically after a new recipe is submitted.

## Context

Depends on 03-01 (Vite scaffold, env vars). Runs in parallel with 03-02 but both modify `App.jsx` ‚Äî this plan finalises `App.jsx` wiring by connecting `SubmitForm`'s `onSuccess` callback to a `refreshTrigger` prop on `RecipeGallery`.

Per the project decision: the frontend reads Supabase directly (not through the backend). The gallery uses the anon key (`VITE_SUPABASE_ANON_KEY`) which is safe to expose in client-side code for public read access.

Supabase `recipes` table columns used: `id`, `title`, `main_category`, `difficulty`, `created_at`.

<interfaces>
Supabase recipes table schema (from server/src/services/supabase.js upsert pattern):

Table: recipes
Relevant columns:
  id            uuid (primary key)
  instagram_url text (unique)
  title         text
  main_category text
  difficulty    text
  created_at    timestamptz (auto, used for ordering)

Query used in gallery:
  supabase.from('recipes').select('id, title, main_category, difficulty').order('created_at', { ascending: false })
</interfaces>

### Task 1: Create Supabase client module and RecipeGallery component

**Action:** Create the shared Supabase client singleton and the `RecipeGallery` component with data fetching and card rendering.

**Files:**
- `client/src/lib/supabase.js`
- `client/src/components/RecipeGallery.jsx`

**Details:**

`client/src/lib/supabase.js`:
```js
import { createClient } from '@supabase/supabase-js';

export const supabase = createClient(
  import.meta.env.VITE_SUPABASE_URL,
  import.meta.env.VITE_SUPABASE_ANON_KEY
);
```

`client/src/components/RecipeGallery.jsx`:

Props:
- `refreshTrigger` (number) ‚Äî incrementing this value causes the gallery to re-fetch. Defaults to `0`.

State:
- `recipes` (array, default `[]`)
- `loading` (boolean, default `true`)
- `error` (string | null, default `null`)

Data fetching:
```js
useEffect(() => {
  async function fetchRecipes() {
    setLoading(true);
    setError(null);
    const { data, error } = await supabase
      .from('recipes')
      .select('id, title, main_category, difficulty')
      .order('created_at', { ascending: false });

    if (error) {
      setError(error.message);
    } else {
      setRecipes(data || []);
    }
    setLoading(false);
  }
  fetchRecipes();
}, [refreshTrigger]);
```

Render:
- While `loading`: `<p className="gallery-loading">Loading recipes...</p>`
- On `error`: `<p className="gallery-error">Failed to load recipes: {error}</p>`
- If `recipes.length === 0` (and not loading): `<p className="gallery-empty">No recipes yet. Submit one above!</p>`
- Otherwise: `<div className="recipe-grid">` containing one `<div className="recipe-card">` per recipe:
  ```jsx
  <div className="recipe-card" key={recipe.id}>
    <h3 className="card-title">{recipe.title}</h3>
    <p className="card-category">{recipe.main_category}</p>
    <p className="card-difficulty">{recipe.difficulty}</p>
  </div>
  ```

Export: `export function RecipeGallery({ refreshTrigger = 0 }) { ... }`
</task>

`client/src/App.jsx` ‚Äî final version wiring both components:
```jsx
import { useState } from 'react';
import { SubmitForm } from './components/SubmitForm';
import { RecipeGallery } from './components/RecipeGallery';

export default function App() {
  const [refreshCount, setRefreshCount] = useState(0);

  const handleSuccess = (recipe) => {
    console.log('New recipe saved:', recipe.title);
    setRefreshCount((c) => c + 1);
  };

  return (
    <div className="app">
      <h1>Recipe Manager</h1>
      <SubmitForm onSuccess={handleSuccess} />
      <RecipeGallery refreshTrigger={refreshCount} />
    </div>
  );
}
```

This is the definitive `App.jsx` ‚Äî it replaces the stub from plan 03-02.
