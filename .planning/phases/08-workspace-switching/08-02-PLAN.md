  <files>client/src/components/LeaveWorkspaceModal.jsx</files>
  <action>
Create `LeaveWorkspaceModal.jsx` — a confirmation modal that handles leaving a workspace, with sole-member detection and workspace deletion.

Props: `{ isOpen, onClose, workspace }` — `workspace` is the workspace object `{ id, name, invite_code }` to leave.

When `isOpen` is false or `workspace` is null, render nothing (return null).

State: `loading`, `error`, `isSoleMember` (boolean | null — null = not yet checked), `checked` (boolean — true once sole-member check is complete).

On mount / when `isOpen` becomes true and `workspace.id` is available, run the sole-member check:
```js
const { count } = await supabase
  .from('workspace_users')
  .select('*', { count: 'exact', head: true })
  .eq('workspace_id', workspace.id);
setIsSoleMember(count === 1);
setChecked(true);
```

Render:
- A fixed overlay with a confirmation card containing:
  - Title: "Leave Workspace"
  - Workspace name displayed
  - If `!checked`: show "Checking…" (while sole-member query runs)
  - If `checked && isSoleMember`: warning paragraph — "You are the only member. **This workspace and all its recipes will be permanently deleted.** This cannot be undone."
  - If `checked && !isSoleMember`: standard message — "You will leave this workspace. You can rejoin with the invite code."
  - Error paragraph (when `error` is set)
  - "Confirm Leave" button (disabled when `loading || !checked`)
  - "Cancel" button (calls `onClose`, resets state)

Leave logic (on "Confirm Leave"):
1. Set `loading: true`, `error: null`.
2. Delete the user's membership row:
   ```js
   await supabase.from('workspace_users').delete()
     .eq('workspace_id', workspace.id).eq('user_id', user.id)
   ```
3. If `isSoleMember`: delete the workspace row:
   ```js
   await supabase.from('workspaces').delete().eq('id', workspace.id)
   ```
4. On any error: set error message, set loading false, return.
5. On success: call `refreshWorkspaces()`. The `refreshWorkspaces` function already handles auto-switching activeWorkspaceId to the first available workspace (or null if 0 remain). Then call `onClose()`.

Do NOT call `setActiveWorkspace` manually after leave — `refreshWorkspaces()` resolves it correctly. `WorkspaceGate` handles the 0-workspace case automatically.

Reset local state (`isSoleMember: null`, `checked: false`, `error: null`, `loading: false`) in a `useEffect` cleanup or whenever `isOpen` transitions to false — so re-opening the modal re-runs the check.

Hooks used: `useAuth()` for `user`, `useWorkspace()` for `refreshWorkspaces`.
